## Technical Decisions and Trade-offs

- **Decision:** Use timestamped markdown files in `dev-log/` rather than a single rolling log file.  
  **Alternatives considered:** A single cumulative file or weekly aggregation files.  
  **Why chosen:** Per-session files preserve chronology cleanly, reduce merge/edit conflicts, and make continuity checks deterministic by filename order.

- **Decision:** Use GMT/UTC timestamp filenames in `YYYY-MM-DD-HHMM.md` format sourced from terminal output.  
  **Alternatives considered:** Local timezone timestamps or manually typed timestamps.  
  **Why chosen:** GMT/UTC removes timezone ambiguity, lexical sort equals chronological order, and terminal-generated values reduce human error.

- **Decision:** Treat chat context as the primary source of truth, using the previous log only to establish the last recorded boundary.  
  **Alternatives considered:** Heavily summarizing the previous log into each new entry.  
  **Why chosen:** This prevents duplication while still enforcing continuity and improves fidelity to actual session events.
