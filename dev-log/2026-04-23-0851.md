## Technical Decisions and Trade-offs

- **Decision:** Build backend before frontend implementation.  
  **Alternatives considered:** Build frontend first with direct source API calls and defer backend.  
  **Why chosen:** Backend is required regardless of frontend design changes and is central to call-efficiency, resilience, and shared data consistency.

- **Decision:** Use overlap windows with dedupe for high-cadence ingestion.  
  **Alternatives considered:** Strict non-overlapping one-minute windows only.  
  **Why chosen:** Overlap reduces data-gap risk from publication latency, clock skew, and transient failures while retaining bounded API usage.

- **Decision:** Choose NestJS (TypeScript) as backend framework.  
  **Alternatives considered:** Express (lighter), other backend stacks.  
  **Why chosen:** Better structured module architecture for ingestion + API layers and stronger portfolio signaling for production-style backend design.
