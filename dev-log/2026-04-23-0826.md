## Technical Decisions and Trade-offs

- **Decision:** Return station-keyed maps instead of full raw arrays for realtime scripts.  
  **Alternatives considered:** Returning all raw rows for each window.  
  **Why chosen:** Map consumption requires one current entry per station; latest-only maps dramatically reduce payload size and simplify downstream UI use.

- **Decision:** Preserve stations with missing data as `null` entries instead of omitting keys.  
  **Alternatives considered:** Dropping stations with no readings in window/sweep.  
  **Why chosen:** Prevents accidental filtering bugs and guarantees complete station visibility for map rendering and diagnostics.

- **Decision:** Keep live integration tests (real API) for script validation.  
  **Alternatives considered:** Mock-only tests.  
  **Why chosen:** Live calls validate true endpoint behavior, pagination warnings, and output-contract correctness against real-world variability.
