## Work Completed

### Portfolio-grade summation of the core backend mechanism
- Captured the agreed mental model in discussion form:
  - **Scheduled ingest** continuously collects marine realtime data and writes **normalized** rows to **PostgreSQL** (system of record, not ephemeral session cache).
  - **Flexibility:** overlap steady state handles jitter; **global watermark** plus **15-minute** chunk backfill handles **server restart**, missed ticks, or **empty database** bootstrap (from **one year** back forward).
  - **Bounded cost:** **rolling one-year retention** keeps storage aligned with the **maximum UI history** and Railway economics.
  - **Serving layer:** REST responses expose **DTOs shaped for the interface** (map snapshot, histories, freshness semantics)—downstream UI consumes **stable contracts**, not raw SWOB GeoJSON passthrough.
- **Why:** Give future implementation and interviews one paragraph that matches how production systems are described—**pipeline + store + read API**—without re-deriving from scattered sections.
- **Outcome:** Clear elevator narrative independent of Nest file layout.

### Frontend vs backend binary split added to project notes
